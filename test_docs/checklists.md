# Чек-листы — Shop API

Чек-листы дополняют детальные тест-кейсы. Смоук используется как быстрая проверка работоспособности после каждого деплоя, регресс — перед релизом.

---

## Смоук-чек-лист

Цель — убедиться, что ключевой функционал не сломан и сервис в принципе работает.

- [ ] `GET /health` возвращает 200, сервис доступен
- [ ] `GET /products` возвращает 200 и непустой список товаров
- [ ] `GET /products/{product_id}` для существующего товара возвращает корректную карточку
- [ ] `POST /orders` с валидными данными создаёт заказ (201)
- [ ] Ответы приходят в формате JSON (`Content-Type: application/json`)
- [ ] Документация API доступна по `/docs`

---

## Регрессионный чек-лист

### Товары (Products)

- [ ] Список товаров возвращается с корректной структурой (`ProductOut`: id, name, price, …)
- [ ] Пагинация `?limit` ограничивает количество элементов
- [ ] Пагинация `?offset` сдвигает выборку
- [ ] Запрос несуществующего товара возвращает 404
- [ ] Запрос с нечисловым `product_id` возвращает 422 (валидация типа)
- [ ] Цена товара — неотрицательное число
- [ ] Поля товара соответствуют контракту по типам

### Пользователи (Users)
    
- [ ] В API нет отдельного ресурса пользователей — `user_id` существует только как поле при создании заказа. Проверки `user_id` вынесены в раздел «Заказы».
### Заказы (Orders)

- [ ] Создание заказа с валидным телом проходит (201)
- [ ] Заказ с несуществующим `product_id` отклоняется
- [ ] Заказ с несуществующим `user_id` отклоняется
- [ ] `quantity = 0` отклоняется (**сейчас БАГ — см. BUG-001**)
- [ ] Отрицательное `quantity` отклоняется (**сейчас БАГ — см. BUG-001**)
- [ ] Заказ количества больше остатка на складе обрабатывается корректно(**сейчас БАГ — см. BUG-002**)
- [ ] Отсутствие обязательного поля → 422
- [ ] Некорректный JSON в теле → 400

### Общие проверки (Common)

- [ ] Несуществующий маршрут → 404
- [ ] Неподдерживаемый HTTP-метод на существующем маршруте → 405
- [ ] Все ответы — валидный JSON
- [ ] Коды ответов соответствуют семантике (2xx/4xx/5xx)
- [ ] Сервис не отдаёт 500 на предсказуемо некорректный ввод (**сейчас БАГ — см. BUG-003**)
